A highly optimized, memory-efficient integration library for pulling, filtering, matching, and copying products and orders from Trendyol and Amazon.

Introduction

When dealing with massive inventories (100k+ products), standard array returns cause fatal memory exhaustion. Marketplace Integrator Pro solves this by using PHP Generators under the hood. It maps vastly different APIs (Trendyol Supplier API and Amazon SP-API) into unified, standardized Models (Product, Order).

Note: This library is strictly designed for reading, listing, and importing data. It is not used for uploading/creating products on marketplaces.

Installation

composer require developertugrul/marketplace-integrator

Initialization & Setup

Initialize your internal mapping and connect your marketplace clients.

Amazon SP-API Requirements

This library natively utilizes highsidelabs/laravel-spapi. Ensure your Laravel .env file includes:

SPAPI_AWS_ACCESS_KEY_ID=
SPAPI_AWS_SECRET_ACCESS_KEY=
SPAPI_LWA_CLIENT_ID=
SPAPI_LWA_CLIENT_SECRET=
SPAPI_LWA_REFRESH_TOKEN=
SPAPI_AWS_ROLE_ARN=
use DeveloperTugrul\MarketplaceIntegrator\MarketplaceManager;
use DeveloperTugrul\MarketplaceIntegrator\ProductMapper;
use DeveloperTugrul\MarketplaceIntegrator\Clients\TrendyolClient;
use DeveloperTugrul\MarketplaceIntegrator\Clients\AmazonClient;
use SellingPartnerApi\Seller\SellerConnector; // Auto-injected in Laravel

// In your Laravel Controller or Command:
public function sync(SellerConnector $amazonConnector) 
{
    // 1. Setup Mapper with your internal DB products
    $mapper = new ProductMapper();
    $mapper->addInternalProduct($internalDbId = 501, $sku = 'TR-TSHIRT-01', $barcode = '869000001');

    // 2. Initialize Manager
    $manager = new MarketplaceManager($mapper);

    // 3. Add Clients
    $manager->addClient(new TrendyolClient('SUPPLIER_ID', 'API_KEY', 'API_SECRET'));
    
    // AmazonClient expects the SellerConnector and defaults to Amazon TR (A33AVAJ2PDY3EV)
    $manager->addClient(new AmazonClient($amazonConnector));
    
    // Ready to sync!
}

Performance: Why Generators?

Instead of waiting for an API to return 50,000 products before you can loop them, our clients yield results immediately as they paginate. Memory consumption remains strictly flat.

Fetching & Filtering Products

Example: Fetching mapped products

// $filters arrays map directly to underlying API query parameters
$filters = ['size' => 200, 'approved' => true];

foreach ($manager->getMappedProducts($filters) as $result) {
    if ($result['is_matched']) {
        echo "Found Match! DB ID: " . $result['internal_id'] . "\n";
    }
    
    $product = $result['marketplace_product']; // Instance of Product Model
    echo $product->title . " - " . $product->price . " TL\n";
}

Order Management & Statuses

Amazon and Trendyol have completely different status names. We map them internally to constants like Order::STATUS_NEW, Order::STATUS_INVOICED, Order::STATUS_SHIPPED, and Order::STATUS_DELIVERED. 

use DeveloperTugrul\MarketplaceIntegrator\Models\Order;

// Filter specifically for newly created orders in the last 24 hours
$filters = [
    'status' => 'Created', // Trendyol specific raw filter
    'startDate' => (new \DateTime('-1 day'))->getTimestamp() * 1000,
];

foreach ($manager->getMappedOrders($filters) as $order) {
    if ($order->status === Order::STATUS_NEW) {
        echo "New Order: {$order->orderNumber} | Customer: {$order->customerFirstName}\n";
        echo "Ship to: {$order->shippingAddress}\n";
        
        foreach ($order->items as $item) {
            echo "-> SKU: {$item->sku} (Mapped to DB ID: {$item->internalProductId})\n";
        }
    }
}

Fast Product Copy (Column Mapping)

Need to quickly import a product from Trendyol to your internal Laravel Database? Use the professional column mapper. It translates the unified Product model into your exact database array structure. 

// The marketplace you want to pull from and the product SKU/Barcode
$marketplace = 'trendyol';
$productId = '8691234567890'; // e.g., barcode

// Define how the library's Product model maps to your Database Columns
$dbColumnMap = [
    'db_title' => 'title',
    'db_price_col' => 'price',
    'stock_amount' => 'quantity',
    'product_sku' => 'sku',
    'brand_name' => 'brand',
    'primary_image' => 'images.0', // Supports dot notation!
];

// Returns a beautiful, ready-to-insert array for your ORM
$insertData = $manager->copyProductToDatabaseFormat($marketplace, $productId, $dbColumnMap);

/* 
Resulting $insertData:
[
    'db_title' => 'Awesome T-Shirt',
    'db_price_col' => 149.90,
    'stock_amount' => 50,
    'product_sku' => 'TSHIRT-01',
    'brand_name' => 'Defacto',
    'primary_image' => 'https://cdn.trendyol.../img.jpg'
]
*/

// Now just insert it into your database!
// Product::create($insertData);
